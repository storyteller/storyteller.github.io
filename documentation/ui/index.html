The Specification Editor and Interactive Runner

Warning! As of now, Storyteller 3.0 is only running on Windows. Before Storyteller 3.0 is officially released, we will also ensure that Storyteller 3.0 is completely usable on OSX and Linux via Mono.

Storyteller comes with an embedded web application to author and interactively execute specifications with a rich client. To launch the application, use the st open command from described below. See Getting Started for more information about the ST.exe tool distributed with Storyteller.

For specific instructions on using the interactive specification tool, see the following topics:

  • The Application Shell
  • The Specification Explorer
  • Working with the Fixture and Grammar Language
  • The Specification Editor

To launch the specification editor from a command line, type the command st open [path] where [path] is the project directory of the specification project you want to load. Other options to control how the testing AppDomain is constructed and other explicit overrides are described below:

st open

st openOpens the specification editor web tool

st open [-b, --build ] [-p, --profile ] [-t, --timeout ] [-l, --lifecycle Acceptance|Regression|Any] [-z, --teamcity] [-c, --config ] [-r, --retries ]
Arguments
pathPath to the StoryTeller project file or the project directory
Flags
[-b, --build <build>]Specify a build target to force Storyteller to choose that profile. By default, ST will use 'Debug'
[-p, --profile <profile>]Storyteller test mode profile for systems like Serenity that use this
[-t, --timeout <timeout>]Optional. Default project timeout in seconds.
[-l, --lifecycle Acceptance|Regression|Any]Optional. Only runs tests with desired lifecyle
[-z, --teamcity]Optional. Applies extra logging to see progress within TeamCity during CI runs
[-c, --config <confi>]Optional. Override the config file selection of the Storyteller test running AppDomain
[-r, --retries <retries>]Sets a minimum number of retry attempts for this execution
